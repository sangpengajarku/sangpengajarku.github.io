3. Instalasi dan konfigurasi web server Apache menggunakan Debian IOS


Jawaban:

Cara Install dan Konfigurasi Web Server (Apache, PHP7, MariaDB) di Debian 9 “Stretch”

Web Server adalah aplikasi server yang sangat populer, Web Server berfungsi menyediakan layanan Web. Untuk menjalankan aplikasi berbasis Web dinamis, setidaknya membutuhkan 3 aplikasi Server, yaitu Web Server (disini menggunakan Apache), Text Processor (disini menggunakan PHP), lalu Database Server (disini menggunakan MariaDB / MySQL), opsional tambahkan Database management (disini menggunakan phpMyAdmin). Asumsinya telah tersedia Komputer Debian 9 yang telah terkonfigurasi IP Address, dan juga telah tersedia Repositori yang Siap digunakan. Berikut akan di bahas satu-persatu cara installasi dan konfigurasinya

1. Installasi Paket-paket Web Server

untuk menginstall paket-paket Web Server, gunakan perintah berikut

apt install apache php7.0 mariadb-server phpmyadmin

Sebeneranya tanpa melalukan konfigurasi apapun, Web Server sudah berjalan dan dapat digunakan / diakses, caranya buka alamat IP Server atau Alamat Domain (jika DNS sudah dikonfigurasi) dari Client.debian9_web_server_1

2. Konfigurasi phpMyAdmin

phpMyAdmin bawaan Debian 9 tidak dapat langsung digunakan, oleh karena itu perlu dilakukan konfigurasi. Lalu secara default MariaDB (MySQL) di Debian 9 tidak memiliki password root (tidak seperti Debian versi sebelumnya), ini juga menjadi salah satu hal yang membuat phpMyAdmin tidak bisa diakses. Berikut adalah konfigurasinya :

mysql -u root -p mysql

Lalu pada Konsol MariaDB, masukan perintah SQL berikut

UPDATE user SET plugin=” WHERE user=’root’;

flush privileges;

exit

debian9_web_server_2

Agar phpMyAdmin dapat menerima akses user root tanpa password, ikuti langkah berikut :

nano /etc/phpmyadmin/config.inc.php

Uncomment baris ini $cfg[‘Servers’][$i][‘AllowNoPassword’] = TRUE;debian9_web_server_3

Keluar dan Simpan, lalu coba buka phpMyAdmin dari client, caranya adalah buka http://alamat_server/phpmyadmin

debian9_web_server_4

Login dengan user root, password default kosongdebian9_web_server_5

3. Mengganti Password root MariaDB (Opsional)

Untuk alasan keamanan, passwword root di MariaDB harus diganti karena defaultnya tidak ada passwordnya. Caranya adalah sebagai berikut :

mysql_secure_installation

Tekan Enter, Enter, sampai pada input New password, masukan password lalu tekan Enter, lalu konfirmasikan password, lalu tekan Enter.

5. Perbedaan konfigurasi debian 9 dan debian 8 itu apa ?


1. Kita harus install paket net-tools

Untuk bisa menjalankan perintah ifconfig dalam debian 9, kita harus menginstall net-tools terlebih dahulu. Saat ingin mengintall bind9 untuk membangun dns server, kita juga membutuhkan paket tersebut.


2. Cara Mengetahui Ip address Debian 9 

Setelah melakukan fresh install debian, umunya kita akan mengecek ip address dengan perintah ifconfig, namun dalam debian 9 belum terdapat paket tersebut, jadi kita menggunakan perintah ip a  atau ip addruntuk melihat ip address kita.


3. Perbedaan Nama Interface 

Pada debian 8, debian 7 dan versi sebelumnya, interface jaringan umumnya menggunakan nama eth0, eth1, dll. Namun, dalam debian 9, default interfacenya menggunakan nama enp0s3. Hal tersebut akan cukup membingungkan jika anda sudah terbiasa menggunakan debian 8.


4. Tidak ada paket Squirrelmail

Setelah saya mencoba untuk membangun webmail server, saya sempat dibingungkan karena tidak ada paket squirrelmail. Sebagai gantinya, kita bisa menggunakan Roundcube untuk membangun webmail server.


5. Paket courier-pop

Saat saya mencoba untuk membuat mail server, dan sebelumnya saya terbiasa menggunakan debian 8, saya juga sempat kebingungan karena tidak ada paket courier-pop, dan saya sempat untuk mengintall paket dovecot. Untuk itu anda harus mengintall courier-base dulu sebelum menginstall courier-pop.

12. Sebutkan konfigurasi DHCP Server pada debian 8.0.


Untuk pengkonfigurasian, langkah-langkahnya sebagai berikut.

Install dulu paket dhcp servernya.

root@debian:~# apt-get install isc-dhcp-server

Konfigurasi DHCP Server Debian 8 (1)

Tekan "Y" untuk melanjutkan.

Setting file konfigurasi DHCP.

root@debian:~# nano /etc/dhcp/dhcpd.conf

Cari kata "# A slightly different configuration for an internal subnet.", dan hapus tanda "#" dari subnet sampai tanda "}"

Konfigurasi DHCP Server Debian 8 (2)

Disini saya menggunakan IP address pada postingan sebelumnya. Untuk keterangan pengaturannya seperti berikut.

Subnet : IP network untuk client.

Netmask : IP netmask untuk client.

Range : jarak IP yang ingin digunakan untuk client.

Option domain-name-servers : IP DNS server.

Option domain-name : domain DNS server.

Option routers : IP DHCP server.

Option-broadcast-address : IP broadcast untuk client.

Kemudian save.

Ubah default interfaces DHCP server menjadi ethernet yang kita gunakan.

root@debian:~# nano /etc/default/isc-dhcp-server

Konfigurasi DHCP Server Debian 8 (3)

Restart DHCP servernya.

Lakukan pengujian pada komputer client, ubah settingan IP menjadi DHCP.

Konfigurasi DHCP Server Debian 8 (4)

Jika otomatis mendapat IP, maka DHCP berhasil

25. File yang digunakan sebagai etak file konfigurasi alamat IP di Debian adalah File yang digunakan sebagai tempat konfigurasi utama dari proxy server adalah File yang digunakan sebagai tempat konfigurasi utama dari mail server adalah File yang digunakan sebagai tempat konfigurasi utama dari DHCP server adalah


file untuk konfigurasi IP address bernama "interfaces", yang terletak di /etc/network/

file untuk konfigurasi proxy bernama "squid.conf", terletak di "/etc/squid/"

kalo untuk mail server, bukan webmail server maka file tersebut bernama "main.cf", filenya terletak di "/etc/postfix/"

file untuk DHCP yaitu "dhcpd.conf", file tsb terletak di "/etc/dhcp3/"

nah itu nama-nama file utama untuk service yang sering digunakan untuk server pada debian, semua file akan ada apabila service tersebut sudah terinstall, jadi pastikan servicenya sudah perinstall terlebih dahulu sebelum mengkonfigurasi semua file tersebut.
